## how to build multiplatform docker

For that you require buildx by default in latest docker 29.0 it is configure by default for older docker you might need to configure driver

Below is the link

https://docs.docker.com/build/building/multi-platform/

Below are the command in use you have docker version below then 29.0 

docker version

docker buildx create \
  --name container-builder \
  --driver docker-container \
  --bootstrap --use


docker buildx ls
docker buildx inspect --bootstrap

#### it will push directly to docker registry #######
#### you can do it without --push but it will save into cached and not show you the docker image using docker image ls ##############

docker buildx build --builder container-builder --platform linux/amd64,linux/arm64  -t test:v1 --push .




##### cache ######

mode=min = cache only the layers that end up in the final image
mode=max = cache all layers, including intermediate build steps

--load means: after the build finishes, import the result into your local Docker image store so it appears in docker images
############


docker buildx build --builder container-builder -t test:v1 --cache-from type=local,src=/tmp/buildx-cache --cache-to type=local,dest=/tmp/buildx-cache,mode=max --load .




##### how to use secret and cache using builtkit in docker ###########

export NPM_TOKEN="your-npm-token"  --> this is env variable created in linux

/root/new.txt my-test-secret-value  this file has value like this that can be used to pass secret 

export DOCKER_BUILDKIT=1    ---> you can do this for the entry but when you need buildkit if you have older docker version in newer version it does not need to enable already enabled

DOCKER_BUILDKIT=1 docker build --secret id=npm_token,env=NPM_TOKEN   --secret id=test_token,src=/root/new.txt   -t my-rocky-app .        ---> it will pass secrets


#### docker build image without cache ########

docker build --no-cache --secret id=npm_token,env=NPM_TOKEN   --secret id=test_token,src=/root/new.txt   -t my-rocky-app .
